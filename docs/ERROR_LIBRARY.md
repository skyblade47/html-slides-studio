### [E-001] TopicInput 组件中函数作用域问题

| 字段 | 内容 |
|------|------|
| **错误ID** | E-001 |
| **发现日期** | 2026-06-05 |
| **严重程度** | 严重 |
| **涉及文件** | `src/components/Editor/TopicInput.tsx` |
| **错误类型** | 逻辑错误 |
| **错误描述** | generateSmartOutline 函数定义在 handleGenerateOutline 内部，导致函数无法被正确调用 |
| **错误原因** | JavaScript 闭包作用域问题，函数在定义前被调用 |
| **修复方案** | 将 generateSmartOutline 函数移到组件外部作为独立函数 |
| **预防措施** | 函数定义应该放在组件外部，或放在组件内部但在使用之前定义 |

## 常见错误分类（HTML 演示项目）

### React 相关错误

| 分类 | 典型错误 | 预防措施 |
|------|---------|---------|
| **Hooks 规则** | useEffect 依赖缺失、条件调用 hooks | 遵循 React Hooks 规则 |
| **状态更新** | 状态更新后立即读取导致读取旧值 | 使用回调或 useEffect 监听 |
| **组件卸载** | 组件卸载后更新状态 | 使用 useRef 或 AbortController |

### TypeScript 相关错误

| 分类 | 典型错误 | 预防措施 |
|------|---------|---------|
| **类型推断** | 隐式 any、类型过宽 | 启用 strict 模式 |
| **接口定义** | Props 不完整、缺少可选字段 | 先定义接口再写组件 |
| **泛型使用** | 泛型约束不正确 | 明确泛型边界 |

### 构建相关错误

| 分类 | 典型错误 | 预防措施 |
|------|---------|---------|
| **依赖问题** | 依赖未安装、版本冲突 | 使用 npm install |
| **路径错误** | 相对路径/绝对路径错误 | 统一使用相对路径 |
| **打包配置** | Vite 配置错误 | 遵循 Vite 最佳实践 |

### HTML 输出相关错误

| 分类 | 典型错误 | 预防措施 |
|------|---------|---------|
| **CSS 冲突** | 动画类名冲突、样式覆盖 | 使用唯一类名前缀 |
| **变量未定义** | CSS 变量未定义 | 检查变量引用 |
| **兼容性** | 使用了不兼容的 CSS 属性 | 使用 Can I Use 检查 |

### 状态管理相关错误

| 分类 | 典型错误 | 预防措施 |
|------|---------|---------|
| **Zustand Store** | Store 初始化问题 | 遵循 Zustand 规范 |
| **状态不一致** | 多个状态源不同步 | 使用单一数据源 |
| **持久化问题** | localStorage 读取失败 | 添加错误处理 |

## 错误库维护指南

### 何时更新错误库

1. **发现新错误时**：记录错误详情和修复方案
2. **修复错误时**：更新预防措施
3. **审查后发现模式时**：总结常见错误模式
4. **项目阶段变化时**：更新错误分类

### 如何使用错误库

1. **开发前**：查阅相关分类，避免常见错误
2. **遇到错误时**：先搜索错误库，看是否已有记录
3. **修复后**：记录到错误库，形成经验积累
4. **审查时**：对照错误库，检查是否重复犯错
