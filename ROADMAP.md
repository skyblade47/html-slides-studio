# HTML Slides Studio - 项目路线图

> AI驱动的HTML演示文稿创作工具

**版本**：v1.0  
**更新日期**：2026-05-29

---

## 项目愿景

打造一款 **AI 驱动** 的 HTML 演示文稿创作工具，让用户通过自然语言即可生成专业级的演示页面，同时保留类似 PPT 的编辑和演示体验。

---

## 版本规划

### 🚧 MVP (v0.1.0) - 最小可行产品
**目标**：验证核心工作流程

- [ ] Tauri + React 项目初始化
- [ ] 项目管理（创建、列表、删除）
- [ ] AI 大纲生成（输入主题 → 输出结构）
- [ ] 内置模板选择（5种预设风格）
- [ ] AI 演示生成（根据大纲+模板生成HTML）
- [ ] 本地 HTML 预览
- [ ] 全屏演示模式

**预计完成时间**：2-3 周

## 技术里程碑

| 阶段 | 里程碑 | 验收标准 |
|------|--------|----------|
| MVP | 可用的最小演示生成器 | 输入主题 → 选模板 → 生成可预览的 HTML |
| v0.2 | 基础编辑能力 | 能手动修改生成的演示 |
| v0.3 | 个性化风格 | 能上传参考图并生成匹配风格 |
| v0.4 | 数据呈现 | 支持常见图表和数据展示 |
| v0.5 | 专业演示 | 提供完整的演讲辅助功能 |
| v0.6 | 便捷分享 | 支持多种分发方式 |
| v1.0 | 稳定可用 | 满足日常使用需求 |

---

## 开发优先级

```
MVP (核心)          →  v0.2 (编辑)    →  v0.3 (审美)    →  v0.4 (图表)
──────────────────────────────────────────────────────────────────────────────
项目管理             拖拽编辑          风格上传           图表组件
AI大纲生成           页面管理           模板管理           AI图表生成
模板系统             撤销/重做         颜色面板           
AI生成HTML           自动保存          预设扩充
本地预览
全屏演示
```

## 资源需求

### 开发者
- 前端开发：1-2 人
- AI 集成：1 人
- 设计/模板：1 人（可兼职）

### 外部依赖
- OpenAI API / Claude API
- Tauri 构建环境
- GitHub Pages / Vercel（可选）

### 工具链
- Node.js 18+
- Rust (for Tauri)
- Git
- 代码编辑器（VS Code 推荐）

---

## 风险与挑战

| 风险 | 影响 | 缓解策略 |
|------|------|----------|
| AI 生成质量不稳定 | 高 | 提供编辑能力作为兜底 |
| HTML 兼容性问题 | 中 | 使用成熟框架，做好测试 |
| 打包体积过大 | 中 | 优化 Tauri 配置，使用 CDN |
| 用户学习成本 | 低 | 预设模板，开箱即用 |
